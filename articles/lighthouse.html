What is Lighthouse? Qu'est-ce que Lighthouse ?

Lighthouse is an automated analysis tool developed primarily for web applications and websites. It is integrated directly into Chrome's developer tools (DevTools), accessible via the dedicated tab, and can also be run from the command line or via a Node.js API. Lighthouse est un outil d'analyse automatisée développé principalement pour les applications et sites web. Il est intégré directement dans les outils de développement du navigateur Chrome (DevTools), accessible via l'onglet dédié, et peut aussi être lancé en ligne de commande ou via une API Node.js.

When an analysis is run, Lighthouse performs a series of automated checks on the target page, then generates a report with scores and actionable recommendations across several dimensions. Lorsqu'une analyse est exécutée, Lighthouse effectue une série de vérifications automatiques sur la page cible, puis génère un rapport contenant des scores et des recommandations concrètes répartis sur plusieurs dimensions.

The limits to understand Les limites à comprendre

This is the most important point: Lighthouse does not measure the overall quality of a website. It measures specific technical aspects according to predefined rules. C'est le point le plus important : Lighthouse ne mesure pas la qualité globale d'un site web. Il mesure certains aspects techniques précis selon des règles prédéfinies.

⚠️ An excellent Lighthouse score does not automatically guarantee that a site is pleasant to use, efficient, or adapted to real user needs. A score of 100 can coexist with a terrible user experience. ⚠️ Un excellent score Lighthouse ne garantit pas automatiquement qu'un site est agréable, efficace ou adapté aux besoins des utilisateurs. Un score de 100 peut coexister avec une très mauvaise expérience utilisateur.

From a QA perspective Du point de vue QA

Lighthouse is a useful tool in a QA toolbox, but it should be treated like any automated test suite: it covers what it can cover, no more. Integrating it into CI gives a reliable baseline and catches regressions on measurable metrics — but it never replaces exploratory testing, user testing, or real-device testing. Lighthouse est un outil utile dans une boîte à outils QA, mais il doit être traité comme n'importe quelle suite de tests automatisés : il couvre ce qu'il peut couvrir, pas plus. L'intégrer en CI donne une baseline fiable et détecte les régressions sur des métriques mesurables — mais il ne remplace jamais les tests exploratoires, les tests utilisateurs, ni les tests sur appareils réels.

💡 Best practice : set score thresholds in your CI pipeline (e.g. Performance ≥ 85, Accessibility ≥ 90) and block deployments that fall below them. 💡 Bonne pratique : définir des seuils de score dans le pipeline CI (ex. Performance ≥ 85, Accessibilité ≥ 90) et bloquer les déploiements qui passent en dessous.

Conclusion Conclusion

Lighthouse is a solid, free, and accessible starting point to audit a web page. Its reports are clear, actionable, and educational. Used regularly, it helps maintain a consistent technical quality baseline. But like any automated tool, it must be combined with human judgment to truly assess the quality of an experience. Lighthouse est un point de départ solide, gratuit et accessible pour auditer une page web. Ses rapports sont clairs, actionnables et pédagogiques. Utilisé régulièrement, il aide à maintenir une baseline de qualité technique constante. Mais comme tout outil automatisé, il doit être combiné avec le jugement humain pour évaluer vraiment la qualité d'une expérience.
